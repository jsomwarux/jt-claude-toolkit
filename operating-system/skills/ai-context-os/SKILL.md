# AI Context OS

Turn company knowledge into agent-usable operating context.

## Positioning
- Sell as an AI implementation foundation, not knowledge-base cleanup.
- Best fit: ops-heavy SMBs where bad context causes missed handoffs, wrong routing, stale answers, approval mistakes, or repeated operator explanations.
- Do not pitch as generic RAG, document chat, Notion cleanup, or file formatting.

## Workflow
1. Pick one target AI workflow first.
2. Inventory where operating knowledge lives.
3. Extract owners, thresholds, exceptions, approvals, sensitive data, escalation paths, examples, and counterexamples.
4. Convert into source-of-truth maps, operating rules, glossary, examples, edge cases, escalation rules, and output expectations.
5. Build an eval pack with 20-50 realistic prompts or tasks.
6. Test before and after context changes.
7. Turn repeated client patterns into reusable vertical IP only after the client-specific version works.

## Quality Gate
- Every context file has a named workflow and source owner.
- Every sensitive action has a human-review or escalation boundary.
- Every reusable rule is backed by client evidence.
- Every sprint ends with eval results, not only documents.
- Public proof remains anonymized unless permission is explicit.
